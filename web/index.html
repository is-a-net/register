is-a.net
使い方 ルール GitHub

あなただけの
.is-a.net を

開発者のための無料サブドメインサービス。
GitHub に PR を送るだけで取得できます。

.is-a.net

使い方

1

フォーク

GitHub リポジトリをフォークしてください。

2

JSON を追加

domains/your-name.json を作成し、DNS レコードを記述します。

3

PR を送信

Pull Request を作成すると自動バリデーションが走り、マージ後に DNS が反映されます。

ドメイン定義ファイル

{
  "owner": {
    "username": "your-github-name",
    "email": "you@example.com"
  },
  "records": {
    "CNAME": "your-name.github.io"
  }
}

ルール

1

1人1サブドメイン
GitHub アカウントにつき1つまで登録可能

a-z

英小文字・数字・ハイフン
2〜63文字。先頭/末尾・連続ハイフン不可

=

PR 送信者 = owner
他人名義の登録は自動で拒否されます

無効なファイル名の例

ファイル名理由
My-Site.json大文字を含んでいる
my_site.jsonアンダースコアは使用不可（ハイフンを使う）
-mysite.json先頭がハイフン
mysite-.json末尾がハイフン
my--site.json連続ハイフン
my.site.jsonサブサブドメインは未対応
12345.json数字のみ
a.json短すぎる（2文字以上必要）
www.jsonシステム予約済み
